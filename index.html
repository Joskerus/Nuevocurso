Tema 5: Tipos de Datos y Operadores: El Lenguaje Secreto de los Datos

R es un lenguaje de programación que permite manejar diversos tipos de datos y aplicar operadores matemáticos y lógicos para procesarlos eficientemente. Comprender estos elementos es fundamental para estructurar y analizar información de manera correcta, detective.

📌 Tipos de Datos en R: Clasificando la Evidencia

R tiene distintos formatos de datos según la información que se maneja. Conocerlos es clave para manipular y analizar correctamente la evidencia, detective.

Vamos a explorar los más utilizados en tu labor de detective.

Tipos de Datos: Números

  • **Números decimales (double)**: Representan valores con punto decimal, como `3.3`, `10.5`, `-2.1`. Piensa en ellos como mediciones exactas.
  • **Números enteros (integer)**: Son valores sin decimales, como `3`, `10`, `-5`. Como el número de casos.
# Ejemplo de double
x <- 3.3
class(x) # Output: "numeric" (R representa 'double' como 'numeric')

# Ejemplo de integer
y <- 3L # La 'L' indica que es un entero
class(y) # Output: "integer"

El tipo `numeric` es el más común para números, `integer` se usa explícitamente.

Tipos de Datos: Texto y Lógicos

  • **Cadenas de texto (character)**: Se usan para almacenar palabras o frases, por ejemplo, `"Municipio"`, `"Sexo"`, `"Nombre"`. Son como las descripciones de las huellas.
  • **Valores lógicos o booleanos (logical)**: Indican verdadero o falso, como `TRUE` o `FALSE`. Útiles para determinar si una pista es real o falsa.
# Ejemplo de character
ciudad <- "Bogota"
class(ciudad) # Output: "character"

# Ejemplo de logical
es_activo <- TRUE
class(es_activo) # Output: "logical"

Los lógicos son esenciales para filtros y condiciones en tus investigaciones, detective.

Tipos de Datos: Fechas y Casos Especiales

  • **Fechas (date)**: Representan días específicos, por ejemplo, `"01/01/2022"`. Cruciales para la cronología de los eventos en tu caso.
  • **Valores especiales**:
    • `NA`: Indica **datos faltantes** (missing). Una pista que no se encontró.
    • `NaN`: Significa "**not a number**", aparece en operaciones erróneas como `0/0`. Un cálculo que no tiene sentido.
    • `Inf`: Se usa cuando el resultado es **infinito**, como `1/0`. Algo que es inmedible.
# Ejemplo de date
fecha_analisis <- as.Date("2023-10-26")
class(fecha_analisis) # Output: "Date"

# Ejemplos de valores especiales
val_faltante <- NA
resultado_error <- 0/0
resultado_infinito <- 1/0

💡 **Consejo, detective:** Es importante identificar el tipo de dato, ya que afectará las funciones de R. ¡Como saber si una huella es de zapato o de bota!

📌 Operadores Aritméticos: Tus Herramientas de Cálculo

Los operadores aritméticos realizan cálculos numéricos básicos, esenciales para cuantificar tus hallazgos:

  • **+**: Suma (`5 + 3` = `8`)
  • **-**: Resta (`10 - 4` = `6`)
  • **\***: Multiplicación (`2 * 6` = `12`)
  • **/**: División (`15 / 3` = `5`)
  • **^**: Potencia (`2 ^ 3` = `8`)
# Ejemplos de operadores aritméticos
suma <- 5 + 3
resta <- 10 - 4
multiplicacion <- 2 * 6
division <- 15 / 3
potencia <- 2 ^ 3

Fundamentales para cualquier cálculo en tu investigación, detective.

Operadores de Comparación: Identificando Coincidencias

Estos operadores comparan dos valores y devuelven un resultado lógico (`TRUE` o `FALSE`). Te ayudan a ver si una pista coincide con lo que buscas:

  • **<**, **>**: Menor o mayor que (`5 < 10` es `TRUE`)
  • **==**: Igualdad (`"Hola" == "Hola"` es `TRUE`)
  • **>=**, **<=**: Mayor o menor igual (`7 >= 7` es `TRUE`)
  • **!=**: Diferente (`"R" != "Python"` es `TRUE`)
# Ejemplos de operadores de comparación
es_menor <- 5 < 10
es_igual <- "R" == "r"
es_mayor_igual <- 7 >= 7
es_diferente <- 10 != 5

Esenciales para filtrar datos y establecer condiciones en tus búsquedas, detective.

Operadores Booleanos: Lógica en tu Investigación

Permiten combinar y manipular valores lógicos. Son tu forma de construir argumentos complejos para tu caso:

  • **&** (AND): Ambos valores deben ser verdaderos para que el resultado sea `TRUE`. Si la huella *y* la coartada son falsas.
  • **|** (OR): Si al menos un valor es `TRUE`, el resultado es `TRUE`. Si el sospechoso estaba *aquí* o *allá*.
  • **!** (NOT): Invierte el valor lógico (`!TRUE` es `FALSE`). Lo opuesto a lo que esperas.
# Ejemplos de operadores booleanos
condicion_and <- (TRUE & FALSE)
condicion_or <- (TRUE | FALSE)
condicion_not <- !TRUE

Permiten construir condiciones complejas para tus análisis, detective.

📌 Operadores de Asignación: La Flecha `<-`

El operador `<-` es el estándar en R para asignar valores a variables. Es como etiquetar tus pruebas para no perderlas de vista, detective:

nombre_persona <- "Laura"
edad_paciente <- 30
suma_valores <- 10 + 5
  • Este operador permite que la variable sea accesible tanto dentro como fuera de funciones.
  • Es la forma preferida para la asignación general en R.

Úsalo para nombrar tus datos y resultados en el expediente del caso, detective.

Operadores de Asignación: El Signo `=` y un Consejo Final

  • **=**: Se usa principalmente para asignar valores **dentro de funciones** (especialmente para argumentos). 
    # Ejemplo de uso de '=' dentro de una función
plot(x = c(1,2,3), y = c(4,5,6), main = "Mi Gráfico")
  • A diferencia de `<-`, la asignación con `=` solo tiene efecto dentro del contexto de la función y la variable mantiene su valor original fuera de ella.

💡 **Consejo, detective:** Para buenas prácticas, usa `<-` en la mayoría de los casos, excepto cuando trabajes dentro de funciones donde `=` es más apropiado. ¡Cada herramienta tiene su lugar en el kit!

🧩 Tema: Vectores y Tablas de Datos en R

En R, la forma en que organizas tus datos es fundamental para tu investigación, detective. Exploraremos dos de las estructuras más importantes: **vectores** y **data.frames** (tablas de datos).

Comprenderlos te permitirá manejar eficientemente cualquier conjunto de datos. ¡Empecemos por la base!

🔹 ¿Qué es un Vector en R?

Un **vector** es la estructura de datos más básica en R. Consiste en una secuencia ordenada de elementos del **mismo tipo** (todos numéricos, todos caracteres, todos lógicos, etc.). Piensa en ello como una lista de huellas del mismo tipo, detective.

Los vectores permiten almacenar datos simples y manipularlos de forma rápida mediante operaciones matemáticas o lógicas.

📌 ¿Cómo se crea un Vector?

Se usa la función `c()` (de “combine”), detective:

# Vector de nombres de enfermedades
enfermedades <- c("COVID-19", "Dengue", "Zika", "Influenza")

# Vector de casos confirmados
casos <- c(1200, 800, 350, 950)

# Vector de estados de vacunación (TRUE = vacunación activa)
vacunacion <- c(TRUE, FALSE, FALSE, TRUE)

💡 Los nombres de variables no deben llevar tildes ni espacios, y los datos tipo texto van entre comillas. ¡Pequeñas reglas para un gran detective!

🔹 ¿Qué es un Data.frame (Tabla de Datos)?

Un **data.frame** es una estructura más compleja que agrupa varios vectores como columnas. Piensa en ello como el expediente completo de un caso, donde cada columna es un tipo de evidencia (edad, diagnóstico, etc.) y cada fila es un caso o paciente individual:

  • Cada columna es una variable (por ejemplo, edad, diagnóstico, vacunación).
  • Cada fila representa una observación (por ejemplo, un paciente o caso individual).
  • Todas las columnas deben tener el mismo número de filas.

📌 Utilidad y Creación de Data.frames

**¿Para qué sirve, detective?**

  • Es la forma principal de trabajar con datos en R, especialmente útil para organizar bases de datos de vigilancia epidemiológica, brotes, encuestas de salud, etc. Es donde archivarás la mayoría de tus casos.

**¿Cómo se crea un data.frame?**

# Tabla con información de cuatro pacientes
datos_pacientes <- data.frame(
  enfermedad = enfermedades,
  casos = casos,
  vacunacion = vacunacion
)

🛠️ Funciones para explorar Data.frames: Tu Lupa del Archivo

Estas funciones permiten conocer las características de un data.frame. Son como las herramientas de tu lupa para examinar los expedientes, detective:

Función Descripción Ejemplo

`rownames()` Devuelve los nombres o identificadores de las filas (los identificadores de los casos) `rownames(datos_pacientes)`
`nrow()` Indica cuántas filas tiene la tabla (el número total de casos) `nrow(datos_pacientes)`
`ncol()` Número de columnas en la tabla (el número de tipos de evidencia) `ncol(datos_pacientes)`
`length()` Devuelve cuántos elementos tiene el objeto (en un data.frame = columnas) `length(datos_pacientes)`

🔧 ¿Qué es una Función, detective?

Una función en R es un conjunto de instrucciones que toma ciertos datos de entrada (input), los procesa mediante operaciones definidas, y devuelve un resultado (output).

Es como una receta para resolver un problema: tú introduces los ingredientes (variables), sigues los pasos (lógica), y obtienes un plato final (resultado). ¡Un truco más en tu arsenal!

Este enfoque permite reutilizar código, automatizar tareas rutinarias y estructurar mejor tus análisis. ¡Eficiencia garantizada!

⚙️ Componentes de una Función: La Anatomía de un Automatismo

Una función en R está compuesta por estas partes clave, como las piezas de un mecanismo de reloj, detective:

  • **Nombre**: cómo se identifica y llama la función. (Su "identidad")
  • **Argumentos**: variables o valores que se entregan como entrada. (Las "pistas" que le das)
  • **Cuerpo (body)**: operaciones que transforman los argumentos. (El "proceso de investigación" interno)
  • **Salida (output)**: el resultado final que se desea obtener, que se devuelve con `return()`. (La "conclusión" de la investigación)

Cada parte es crucial para el funcionamiento de la "receta".

📦 Ejemplo Clásico: Cálculo de IMC (Índice de Masa Corporal)

Usamos el ejemplo del Índice de Masa Corporal (IMC), cuyo cálculo es una medida estándar en salud. La fórmula es:

\[ \text{IMC} = \frac{\text{peso (kg)}}{\text{talla (m)}^2} \]

IMC <- function(peso, talla) {
  resultado <- peso / (talla^2)
  return(resultado)
}

Una función simple para un cálculo común que puedes automatizar, detective.

Calculando el IMC: Poniendo la Función en Acción

Si calculamos el IMC para una persona de 80 kg y 1.75 m. Observa la facilidad, detective:

IMC(peso = 80, talla = 1.75)
# [1] 26.12245

Así es cómo se usa la función que acabamos de definir. ¡Una pieza de tu caso resuelta con un comando!

🏥 Nuevo Ejemplo en Salud Pública: Tasa de Ataque

La tasa de ataque es una medida clave durante brotes epidémicos. Nos ayuda a entender qué tan rápido se propaga una enfermedad. Se calcula como:

\[ \text{Tasa de ataque} = \frac{\text{número de casos nuevos}}{\text{población en riesgo}} \times 100 \]

tasa_ataque <- function(casos, poblacion) {
  resultado <- (casos / poblacion) * 100
  return(resultado)
}

Automatizando un cálculo epidemiológico esencial, detective.

Calculando la Tasa de Ataque: Obteniendo el Resultado de tu Pista

Y utilizarla para calcular una tasa. Si tienes 25 casos en una población de 200 personas, detective:

tasa_ataque(casos = 25, poblacion = 200)
# [1] 12.5

Esta función puede integrarse fácilmente en análisis más grandes o aplicarse a múltiples localidades. ¡Multiplica tu capacidad de investigación!

🔗 Uso del Operador PIPE `%>%`: La Lógica del Detective (Sin PIPE)

El operador tubería (`%>%`) es una herramienta clave de Tidyverse que encadena funciones, evitando que el código sea difícil de leer. Compara esta forma de escribir un análisis; es como intentar seguir varias pistas desorganizadas, detective:

datos_filtrados <- filter(dat, edad > 18)
datos_con_esquema <- mutate(datos_filtrados, esquema = ifelse(dosis > 2, "Completo", "Incompleto"))
datos_agrupados <- group_by(datos_con_esquema, esquema)
datos_por_esquema <- summarise(datos_agrupados, personas = n())

El código puede volverse complejo y difícil de seguir, como un laberinto de pistas.

🔗 Uso del Operador PIPE `%>%`: La Lógica del Detective (Con PIPE)

**Con PIPE (`%>%`) – Más legible y eficiente:** Ahora, mira cómo el operador `%>%` te permite encadenar tus acciones de investigación de forma fluida, como un plan de ataque perfectamente ejecutado, detective:

datos_por_esquema <- dat %>%
  filter(edad > 18) %>%
  mutate(esquema = ifelse(dosis > 2, "Completo", "Incompleto")) %>%
  group_by(esquema) %>%
  summarise(personas = n())

📌 **Ventajas de `%>%`:**

  • ✔️ Hace el código más legible.
  • ✔️ Reduce líneas innecesarias.
  • ✔️ Encadena funciones sin necesidad de múltiples asignaciones.

Un cambio pequeño, un gran impacto en la legibilidad de tu investigación.

🔍 Funciones esenciales de Tidyverse: `glimpse()` y `summarise()`

  • **1️⃣ `glimpse()` – Exploración rápida de datos**

    Muestra un resumen general del data.frame: número de filas, columnas, nombres de variables y tipos de datos. ¡Es como tu primera mirada rápida a la escena del crimen, detective!

    dat %>% glimpse()
  • **2️⃣ `summarise()` – Estadísticas resumidas**

    Calcula métricas como la media de edad y el total de registros en la tabla. Ideal para obtener las cifras clave de tu caso.

    dat %>% summarise(media_edad = mean(edad), casos_totales = n())

🔍 Funciones esenciales de Tidyverse: `group_by()`

  • **3️⃣ `group_by()` – Agrupación de datos**

    Para análisis epidemiológicos, podemos agrupar datos por una variable, como sexo, y calcular indicadores por grupo. Es como clasificar a los sospechosos por características comunes, detective.

    dat %>%
  group_by(sexo) %>%
  summarise(casos = n(), promedio_edad = mean(edad))

🔍 Funciones esenciales de Tidyverse: `select()` y `filter()`

  • **4️⃣ `select()` – Extraer columnas específicas**

    Si solo quieres trabajar con ciertas variables, como edad y sexo. Es como centrarte solo en las pistas relevantes para el caso, detective.

    dat %>% select(edad, sexo)
  • **5️⃣ `filter()` – Filtrar registros según criterios**

    Para obtener registros de pacientes menores de 28 años y sexo femenino. Esto te permite aislar los casos de interés para una investigación más profunda.

    dat %>% filter(sexo == "F", edad <= 28)

🔍 Funciones esenciales de Tidyverse: `arrange()`

  • **6️⃣ `arrange()` – Ordenar registros**

    Ordena datos por edad de menor a mayor. Es como organizar tus expedientes cronológicamente o por importancia, detective.

    dat %>% arrange(edad)

    Para ordenarlo de mayor a menor, usamos `desc()`:

    dat %>% arrange(desc(edad))

🔍 Funciones esenciales de Tidyverse: `mutate()` y `rename()`

  • **7️⃣ `mutate()` – Crear nuevas variables**

    Si quieres agregar una nueva columna con información adicional. Como añadir una nueva categoría a tu análisis de pistas, detective.

    dat %>% mutate(dpto = toupper(nombre_departamento))
  • **8️⃣ `rename()` – Modificar nombres de variables**

    Si quieres renombrar una columna para facilitar la interpretación. Como cambiar el nombre de un código secreto a algo más claro para tu informe, detective.

    dat %>% rename(departamento = nombre_departamento)

🔍 Funciones esenciales de Tidyverse: `slice()`

  • **9️⃣ `slice()` – Extraer filas específicas**

    Para seleccionar un rango de registros, por ejemplo de la fila 10 a la 15. Esto te permite enfocarte en un segmento particular de la evidencia, detective.

    dat %>% slice(10:15)
